What is File Path ?

A file path is the unique address for a file or folder within a computer's file system, specifying its exact location by listing the sequence of directories leading to it.

File paths are essential for accessing, organizing, and manipulating files and folders, with two main types:
• Absolute paths : which start from the file system's root directory (e.g., C:\Users\Documents\file.txt in Windows)
• Relative paths : which specify a location in relation to the current working directory (e.g., ../images/picture.jpg).

Why we can't provide Absolute Path of the File in HTML Doc ?

Browsers implement security measures to prevent web pages from directly accessing local file systems for security and privacy reasons. If an HTML file could arbitrarily access any file on your computer via an absolute path, it would pose a significant security risk.

What is BoilerPlate Code ?

An HTML boilerplate provides the basic structure and essential elements required for any HTML5 webpage. This foundational code ensures consistency and saves time during web development.

<!DOCTYPE html>:

This declaration informs the browser that the document is an HTML5 document.

<html lang="en">:

This is the root element of the HTML page, and the lang="en" attribute specifies the language of the document as English, which is beneficial for accessibility and search engines.

<head>:

This section contains meta-information about the HTML document, which is not directly visible on the webpage but is used by browsers and search engines.

  • <meta charset="UTF-8">:

    Specifies the character encoding for the document, ensuring proper display of various characters.

  • <meta name="viewport" content="width=device-width, initial-scale=1.0">:

    This meta tag is crucial for responsive web design, instructing the browser on how to control the page's dimensions and scaling for different devices.

  • <title>Document</title>:

    Sets the title that appears in the browser tab or window title bar.

<body>:

This section contains all the visible content of the webpage, such as text, images, links, and other elements that users interact with.

UTF-8

UTF-8 (Unicode Transformation Format – 8-bit) is the most common character encoding for the World Wide Web and a standard for representing all Unicode characters. It is a variable-length encoding that uses one to four bytes to represent each character, with more frequently used characters (like ASCII) using fewer bytes to save space. UTF-8 is backward-compatible with ASCII and can represent characters from virtually all written languages and symbols, making it essential for modern digital communication.

ASCII

ASCII, or American Standard Code for Information Interchange, is a character encoding standard that assigns numerical values to letters, numbers, and symbols to enable electronic communication between computers. It uses a 7-bit binary system to define 128 characters, including printable characters like digits and letters, and non-printable control characters for formatting. While the original standard is 7-bit, modern systems often store these in 8-bit bytes, and the foundational 128 ASCII characters serve as the basis for more extensive encoding systems like Unicode.

HTML Structural Tags :D

  • <header>

    The <header> tag is a semantic HTML element that is used to define the introductory or navigational content of a webpage or a section.

  • <main>

    It defines a document's main content, which should be unique. It excludes content like sidebars, navigation, logos, and copyright info, ensuring unique document-specific material within.

  • <footer>

    The <footer> tag in HTML is used to define the footer section of an HTML document. The footer section typically contains information such as contact information, sitemap, back-to-top links, related documents, copyright, etc. The footer tag is a semantic tag included in HTML

  • <div>

    The <div> tag, used in over 90% of web pages, divides content into sections and serves as a flexible container for text, images, and links.
